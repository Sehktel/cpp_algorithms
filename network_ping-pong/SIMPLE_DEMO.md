# Простая демонстрация Ping-Pong TCP клиент-сервер

## Описание
Упрощенная версия TCP клиент-сервер приложения для демонстрации на доске.
Только для Linux, без кроссплатформенности.

## Структура проекта
```
network_ping-pong/
├── client/
│   ├── simple_client.cpp    # Упрощенный клиент
│   └── client.cpp           # Полная версия
├── server/
│   ├── simple_server.cpp    # Упрощенный сервер  
│   └── server.cpp           # Полная версия
└── Makefile                 # Простая сборка
```

## Компиляция и запуск

### 1. Сборка
```bash
# Сборка всех программ
make all

# Или по отдельности
make simple_server
make simple_client
```

### 2. Запуск
```bash
# Терминал 1 - Запуск сервера
./simple_server

# Терминал 2 - Запуск клиента  
./simple_client
```

### 3. Очистка
```bash
make clean
```

## Что происходит

### Сервер:
1. Создает сокет
2. Привязывает к порту 8080
3. Переводит в режим прослушивания
4. Принимает подключение клиента
5. Получает сообщения и отвечает:
   - "ping" → "pong"
   - "exit" → завершение работы

### Клиент:
1. Создает сокет
2. Подключается к серверу (localhost:8080)
3. Отправляет 3 сообщения "ping"
4. Получает ответы "pong"
5. Отправляет "exit" и завершается

## Ключевые концепции для объяснения

### Сокеты
- `socket()` - создание сокета
- `bind()` - привязка к адресу
- `listen()` - режим прослушивания
- `accept()` - принятие подключения
- `connect()` - подключение к серверу
- `send()`/`recv()` - отправка/получение данных
- `close()` - закрытие сокета

### Структуры
- `sockaddr_in` - адрес IPv4
- `AF_INET` - семейство адресов IPv4
- `SOCK_STREAM` - тип сокета TCP
- `htons()` - преобразование порядка байт

### Протокол
- TCP - надежная доставка данных
- Порты - идентификаторы сервисов
- localhost (127.0.0.1) - локальный адрес

## Преимущества упрощенной версии

1. **Меньше кода** - легче писать на доске
2. **Только Linux** - нет условной компиляции
3. **Прямолинейная логика** - без сложных функций
4. **Минимум зависимостей** - только стандартные библиотеки
5. **Понятные комментарии** - на русском языке

## Альтернативные подходы

### Вместо упрощения можно было:
1. **Использовать библиотеки** (boost::asio, libevent)
   - Плюс: меньше кода
   - Минус: сложнее объяснить концепции

2. **Асинхронное программирование** (epoll, select)
   - Плюс: более эффективно
   - Минус: сложнее для понимания

3. **Многопоточность** (threads)
   - Плюс: обработка нескольких клиентов
   - Минус: усложняет демонстрацию

### Выбранный подход:
- **Синхронный** - проще для понимания
- **Один клиент** - фокус на основах
- **Минимум абстракций** - видно все детали
