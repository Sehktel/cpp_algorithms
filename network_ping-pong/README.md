# Ping-Pong Сетевое Приложение на C++

## Описание
Учебный проект для изучения основ сетевого программирования на C++. Простое клиент-серверное приложение, где клиент отправляет "ping", а сервер отвечает "pong".

## Структура проекта
```
├── server/
│   └── server.cpp          # Серверное приложение
├── client/
│   └── client.cpp          # Клиентское приложение
├── CMakeLists.txt          # Конфигурация сборки
└── README.md               # Документация
```

## Что изучается в этом проекте
1. **Сокеты (Sockets)** - основа сетевого взаимодействия
2. **TCP протокол** - надежная передача данных
3. **Клиент-серверная архитектура**
4. **Синхронный обмен сообщениями**

## Требования
- C++17 или выше
- CMake 3.10+
- Windows/Linux/macOS

## Сборка проекта

### Windows
```bash
mkdir build
cd build
cmake ..
cmake --build .
```

### Linux/macOS
```bash
mkdir build
cd build
cmake ..
make
```

## Запуск

### 1. Запустить сервер
```bash
# Windows
.\build\Debug\ping_pong_server.exe

# Linux/macOS
./build/ping_pong_server
```

Сервер запустится на порту **8080** и будет ожидать подключения.

### 2. Запустить клиент (в другом терминале)
```bash
# Windows
.\build\Debug\ping_pong_client.exe

# Linux/macOS
./build/ping_pong_client
```

Клиент подключится к серверу, отправит несколько сообщений "ping" и получит ответы "pong".

## Как это работает

### Сервер
1. Создает сокет для прослушивания входящих соединений
2. Привязывает сокет к IP адресу и порту
3. Ожидает подключения клиента
4. Получает сообщения от клиента
5. Если получено "ping", отправляет обратно "pong"
6. Продолжает работу до получения "exit"

### Клиент
1. Создает сокет для подключения
2. Подключается к серверу по IP и порту
3. Отправляет несколько сообщений "ping"
4. Получает и выводит ответы от сервера
5. Отправляет "exit" для завершения

## Модификации для изучения

### Базовые
- Изменить порт сервера
- Изменить количество ping-сообщений
- Добавить timestamp к сообщениям

### Продвинутые
- Добавить поддержку нескольких клиентов (многопоточность)
- Реализовать UDP версию (вместо TCP)
- Добавить шифрование сообщений
- Создать простой протокол с командами
- Измерить задержку (latency) между ping и pong

## Основные концепции

### TCP Socket
TCP (Transmission Control Protocol) обеспечивает:
- ✅ Надежную доставку данных
- ✅ Гарантированный порядок сообщений
- ✅ Контроль перегрузки
- ❌ Больше накладных расходов чем UDP

### Последовательность работы с сокетами

**Сервер:**
```
socket() → bind() → listen() → accept() → recv()/send() → close()
```

**Клиент:**
```
socket() → connect() → send()/recv() → close()
```

## Возможные ошибки и решения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| "Address already in use" | Порт занят | Закрыть предыдущий процесс или изменить порт |
| "Connection refused" | Сервер не запущен | Сначала запустить сервер |
| "Connection timeout" | Проблемы с сетью/firewall | Проверить firewall и сетевые настройки |
